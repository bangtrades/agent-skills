# Nova Kids Beta Readiness Visual Criteria

## Home

Beta standard:

- Child understands chalkboard, bookshelf, trophy case, and Dashy's desk without reading.
- Objects feel physically located in the room.
- Tap affordances are visible but not like debug overlays.

## Lessons

Beta standard:

- Child can tap one obvious control to hear the page.
- Workbook shell feels physical.
- Each page type has a distinct classroom material:
  - Story: picture book page.
  - Concept: chalk diagram or explainer note.
  - Quiz: magnetic tiles or sticky notes.
  - Experiment: tabletop materials.
  - Voice: microphone/listening object.
- Page turns and progress are understandable without adult explanation.

## Library

Beta standard:

- Lessons feel like books on shelves.
- New, completed, locked, and empty states are visible as classroom objects.
- Generated lessons can use placeholder book covers without looking broken.

## Trophies

Beta standard:

- Trophy page feels like a trophy case on a wall.
- Empty achievement slots look aspirational, not broken.
- Completion moment feels celebratory and connected to the lesson.

## Dashy

Beta standard:

- Dashy has pose/state variety: idle, pointing, reading, listening, thinking, celebrating.
- Dashy helps the child understand what to do next.
- Dashy does not cover primary controls.

## Onboarding and Auth

Beta standard:

- Entry feels like entering the classroom.
- Parent gates are clearly adult-only without scaring the child.
- Child profile choices feel like stickers/name tags.

## Loading, Error, and Offline

Beta standard:

- Loading is in-world: chalk spinner, thinking Dashy, classroom prep.
- Errors are friendly notes or signs.
- Offline state explains recovery without sounding technical.
