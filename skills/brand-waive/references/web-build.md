Pairs with `frontend-design`. This is the exact page shell the production site uses ‚Äî clone it for
any new page, article, landing page, or HTML deliverable. Static, hand-written, no framework
required (the site is static HTML/CSS/JS on Vercel with `cleanUrls`).

**Critical ordering:** load `site.css` AFTER the page's inline `<style>` so its custom-property
overrides re-skin the page for dark mode without rewriting layout CSS. Define every surface color as
a `var(--token)` (see `palette.md`) ‚Äî hardcoded `#fff`/white literals won't theme and will glow in
dark mode.

## Layout system

- Container: `width: min(1180px, calc(100% - 48px)); margin-inline: auto;` (articles use ~1080px).
- Fixed glass header (`backdrop-filter: blur(18px)`), `--header-bg` token.
- Cards: `--card-bg` / `--card-bg-strong`, `border: 1px solid var(--line)`, radius 16‚Äì24px,
  `--shadow-soft`. Generous padding (28‚Äì42px on feature blocks).
- Buttons: pill (`border-radius: 999px`). Primary = blue gradient; CTA = orange gradient; ghost =
  `--blue-deep` text + hairline border. One orange CTA per view.
- Eyebrow ‚Üí H1 ‚Üí lede is the standard section opener.
- Responsive: collapse multi-column ‚Üí single at ~900‚Äì980px; hide secondary nav links ‚â§720px
  (`.site-nav a:not(.nav-build){display:none}`).
- Respect `prefers-reduced-motion` (collapse animations to final state).

## Accessibility (ships site-wide ‚Äî match it)

Skip link on every page; visible `:focus-visible` ring (`--focus-ring`); semantic `<button>` for
interactive elements (never clickable `<div>`); body-prose links underlined (non-color signifier);
state never by color alone (add ‚úì/‚öÝ/‚úï glyphs); `<img>` carries `alt` + `width`/`height`.

## Interactive demos (the "Lab" idiom)

Dark consoles (`#0b1322` / `#121d33` panels, `#6f87b5` eyebrows) embedded in light editorial pages =
the WaiveLabs Lab signature. Hand-rolled SVG charts (no chart libs), fake/sample data only, a
"Demo ¬∑ sample data" badge inside every console, ES5-style namespaced IIFEs. Dark consoles are
dark-native ‚Äî they do NOT re-skin in light mode; only the page scaffold themes.

## SEO + AEO

One `<h1>` per page; descriptive `<title>` ending `¬∑ WaiveLabs`; JSON-LD where it fits
(Organization on home, BlogPosting on articles, FAQPage + BreadcrumbList on `/for` pages); keep
indexable marketing pages in `sitemap.xml`; Lab/demo pages are `noindex`. Plain-English first
paragraphs (answer-engine friendly); the site ships `llms.txt`.
