# Client Pitch Visual System

## Brand Adaptation

Read the client brand skill before making visual decisions. Translate the narrative into the client’s:
- colors
- typography
- icon style
- photography/product language
- executive voice
- density level

Do not treat the Summer Fridays deck as a design template. Treat it as a narrative template.

## Imagegen Usage

Use imagegen for:
- messy current-state operating model
- clean future-state operating system
- acceleration analogy
- client-specific product/dashboard hero visuals

Prompt generated images as text-free, brand-compatible bitmaps. Add all labels in editable slide text.

Image prompt pattern:

```text
Use case: productivity-visual
Asset type: PowerPoint slide visual, text-free editorial illustration
Primary request: <client-specific visual metaphor>
Brand style: <tokens from client brand skill>
Composition: 16:9, leave whitespace for headline and editable labels
Avoid: readable text, fake logos, brand marks, watermarks, neon, generic AI robots
```

## Diagrams

Use editable shapes for architecture, timeline, and scenario slides.

Architecture rules:
- Data flows left to right.
- The connector layer visibly feeds the structured data layer.
- The intelligence loop on the right must feel continuous: app layer, features, memory, back to app layer.
- Use bands, loops, or clean staged lanes instead of many thin crossing lines.
- Labels must anchor to the thing they describe.

## Copy Density

Default maximums:
- Cover: 1 headline, 1 subtitle, 2 pills.
- Explainer slides: 1 claim, 1 diagram.
- Use case slides: 4-5 cards, each with one outcome and one short proof line.
- Roadmap/scenarios: concise lists only.

If text shrinks below readable presentation size, reduce content rather than shrinking further.

## Visual Rhythm

Use varied macro layouts:
- cover with proof visual
- radial explainer
- image-led current-state pain
- architecture map
- card row for use cases
- analogy visual
- tier matrix
- timeline
- close model

Avoid three consecutive card-grid slides.
