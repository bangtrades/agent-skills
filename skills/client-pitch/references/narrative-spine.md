# Client Pitch Narrative Spine

Use this as the default 8-9 slide structure. Preserve the arc, but adapt the examples, labels, and design system to the client.

## Slide 1 — Cover / Thesis

Purpose: establish the proposed transformation and show a credible artifact or hero visual.

Pattern:
- Title: `[Client] Agentic AI Growth / Operating System / Transformation`
- Subtitle: one sentence on moving from manual work to agent-led execution.
- Visual: client-specific product/app/dashboard preview or polished conceptual image.
- Pills: offer label and audience context, such as `Agentic AI`, `Executive demo`, `90-day sprint`.

## Slide 2 — What Agents Are

Purpose: explain agents simply.

Core claim: `Agents are AI coworkers.`

Visual:
- Center circle: `Agents`
- Four equal-distance capability boxes: `Observe`, `Reason`, `Act`, `Remember`
- Short sublabels only.

Avoid long definitions. The presenter will explain.

## Slide 3 — Current Operating Mess

Purpose: make the pain visible.

Show the client’s current state as a messy but elegant operating model:
- spreadsheets
- manual reporting
- channel or department silos
- Slack/email decisions
- no shared memory
- late signals
- lots of tools

Use a text-free generated background or editable collage, with short editable labels.

## Slide 4 — System Architecture

Purpose: show the solution is real infrastructure, not a chatbot.

Default architecture:
- Left: data sources
- Middle-left: connector/ELT layer such as Coupler.io, Fivetran, Airbyte, Zapier, or native APIs
- Middle: structured data layer such as Supabase, BigQuery, Snowflake, Postgres
- Right: app layer, memory layer, and agent/application features

The right side should show a loop:
`app layer -> Growth Desk/features -> memory -> app layer`

Use smooth flow bands or clean loop geometry. Avoid ugly connector clutter.

## Slide 5 — Use Cases

Purpose: make agent value concrete.

Use 4 workstreams. Localize to client:
- Campaign cockpit
- Forecast planner
- Creator/customer/account scout
- Competitor/category radar
- Customer service QA
- Retail/channel operations
- Product launch command center
- Finance close assistant

Each card should have one verb-led outcome, not a paragraph.

## Slide 6 — Acceleration Analogy

Purpose: make the roadmap memorable.

Default car analogy:
- `Today`: `10 MPH`, `basic GPT`
- `60 Days`: `50 MPH`, `10 agents`
- `180-365 days`: `150 MPH`, `25 agents`

Add the claim: every model release upgrades the whole stack.

Adapt the metaphor when cars are off-brand. Options: runway, flywheel, production line, studio, kitchen, trading desk, or lab.

## Slide 7 — Build Scenarios

Purpose: turn the analogy into buying options.

Default tiers:
- **Tier 1 / Lean build:** data collection and structuring, basic memory such as Pinecone/vector DB, custom skills, custom software development, agent workflows, client apps + Codex on desktops, personnel training, future Agent Director handoff.
- **Tier 2 / Control plane:** agentic control plane such as Paperclip or an alternative such as LangGraph Studio, memory system such as GBrain or similar, API-powered agents, agents capable of extending the platform, context engineering, specification engineering.
- **Tier 3 / Enterprise:** 12-18 month horizon, LangChain/LangGraph, self-improving agent harnesses and skills, dozens of specialist agents across departments, seamless AI workflows, mature application layer replacing vendor sprawl, fully owned by the client.

Make it clear which tier is recommended.

## Slide 8 — Roadmap / Timeline

Purpose: show the next decision.

Default phases:
- `0-30`: Foundation
- `31-60`: Automation
- `61-90`: Scale

Use client-specific phase outputs. Include training, agent workflows, memory, approval gates, and human-gated decisioning where relevant.

## Slide 9 — Close / Operating Model

Purpose: ask for the engagement.

Pattern:
- One strong close claim.
- Three roles: `Agent Director`, `Client team`, `Agent fleet`
- Final next step: start the build sprint or approve the pilot.

Keep it confident and plain.
