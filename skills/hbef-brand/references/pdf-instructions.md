# HBEF PDF Instructions

PDFs for HBEF should be generated by one of two paths:

1. **Word → PDF export** (when content is text-heavy: letters, memos, annual reports, IC briefs). Use `references/docx-instructions.md` to build the Word doc, then export.
2. **HTML → PDF render** (when content is visual: one-pagers, infographics, dashboards, sponsor prospectuses). Use the `pdf` skill to render branded HTML.

This file covers the HTML→PDF path specifically. For Word→PDF, follow `docx-instructions.md` and use the resulting docx export.

## Page setup (HTML→PDF)

- **Page size:** US Letter (8.5" × 11") portrait; 11" × 8.5" landscape for dashboards
- **Margins:** 0.5" top/bottom, 0.6" left/right (tighter than Word — PDFs have less reading distance)
- **Default font stack:** `'Montserrat', 'Avenir Next', 'Helvetica Neue', Arial, sans-serif`
- **For HBEF cover pages and display headlines**, embed BauhausBold from `assets/fonts/bauhaus-bold.woff` via `@font-face`

## Running header & footer (HTML→PDF)

Use `@page` rules to define repeating margins:

```css
@page {
  size: letter;
  margin: 0.6in 0.6in 0.7in 0.6in;
  
  @top-left {
    content: "Q1 2026 Endowment Report";
    font-family: 'Montserrat', sans-serif;
    font-size: 8pt;
    font-weight: 700;
    color: var(--hbef-navy);
    text-transform: uppercase;
    letter-spacing: 0.5pt;
  }
  
  @top-right {
    content: "Confidential — Board Working";
    font-family: 'Montserrat', sans-serif;
    font-size: 8pt;
    font-style: italic;
    color: var(--ink-60);
  }
  
  @bottom-left {
    content: "Hermosa Beach Education Foundation · hbef.org";
    font-family: 'Montserrat', sans-serif;
    font-size: 8pt;
    color: var(--ink-60);
  }
  
  @bottom-right {
    content: "Page " counter(page);
    font-family: 'Montserrat', sans-serif;
    font-size: 8pt;
    font-weight: 700;
    color: var(--hbef-navy);
  }
}

@page :first {
  /* Cover has no header/footer */
  @top-left { content: none; }
  @top-right { content: none; }
  @bottom-left { content: none; }
  @bottom-right { content: none; }
  margin: 0;
}
```

## Recommended toolchain

For HTML→PDF rendering, the `pdf` skill supports several backends. For HBEF deliverables, prefer (in order):

1. **Playwright + headless Chromium** — best CSS support including `@font-face`, `@page` margin boxes, advanced layout
2. **WeasyPrint** — pure-Python, faithful CSS-Paged-Media support; no JS execution
3. **wkhtmltopdf** — fast, but older CSS engine; avoid unless others fail

When using Playwright, set `print_background=True` so colored bands (cover navy, teal footer) render correctly.

## Final checks (before sharing)

1. Cover renders correctly with navy band + lockup centered.
2. BauhausBold or fallback rendering verified (open PDF and inspect headlines).
3. Page numbers paginate (verify on a multi-page render).
4. Tax language footer present on donor-facing deliverables.
5. Colors render in print preview — print-bg flag set, not stripped.
6. File size reasonable (< 5MB for typical 1–10 page report); compress images if oversized.
7. Embed fonts in PDF metadata if recipient may not have Montserrat installed.
